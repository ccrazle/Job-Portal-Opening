Make this a skill by keeping some design principles in mind.

Key design principles to keep in mind:

- Progressive disclosure — Keep the SKILL.md under ~500 lines. Put detailed reference material in separate files and point to them.
- Explain the "why" — Instead of heavy-handed "you MUST do X," explain why something matters. This generalizes better.
- Use imperative form — Write instructions as direct commands ("Extract the text," "Validate the output").
- Domain organization — If the skill supports multiple variants (e.g., AWS vs GCP vs Azure), organize references by variant so only the relevant one gets loaded.
- Test cases — For skills with objectively verifiable outputs, pair them with test prompts and assertions to iterate on quality.
